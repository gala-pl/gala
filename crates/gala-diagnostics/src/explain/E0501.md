# E0501: Cannot Auto-Uncompute

The compiler cannot automatically synthesize an uncomputation for this value.

## Example

```gala
fn main() -> Qubit quantum {
    let q = qubit();
    let r = measure(q);  // r is classical, q is consumed
    // q is already gone, but if we had an ancilla depending on r...
    drop(q);  // error: cannot uncompute, depends on measurement
}
```

## Explanation

Automatic uncomputation only works for *liftable* (qfree) values — values computed purely by reversible operations from still-live inputs. If a value depends on a measurement result or other irreversible operation, the compiler cannot synthesize its adjoint.

## Fix

Provide an explicit uncomputation strategy:

```gala
fn main() -> Qubit quantum {
    let q = qubit();
    let r = measure(q);
    uncompute_with(q, manual_reverse_circuit(r));  // explicit
}
```

Or restructure to avoid the dependency.
